## 写作方法论

### 穿越爽文大纲构建
**来源**: 《穿越之我在古代搞钱》项目
**时间**: 2026-02-24

| 问题 | 答案 |
|------|------|
| **输入** | 穿越爽文需求，目标100万字+，古代发家致富主题 |
| **输出** | 五卷大纲 + 主要人物卡 + 世界观文档 |
| **不变量** | 1. 先定核心冲突 2. 再铺世界规则 3. 最后填人物动机 |
| **可变参数** | 爽度配比、金手指强度、历史考据严格度 |
| **失败点** | 大纲太死没有弹性、人物动机不足、金手指太bug |

## 记忆系统运维

### 消息去重机制
**来源**: 飞书消息重复问题排查
**时间**: 2026-02-25

| 问题 | 答案 |
|------|------|
| **输入** | 同一消息多次触发的问题报告 |
| **输出** | 找到根本原因（历史遗留+重复配置），完成修复 |
| **不变量** | 1. 先诊断再修复 2. 配置>代码 3. 重启验证 |
| **可变参数** | 检查频率、通知方式、回滚策略 |
| **失败点** | 没看网关日志就猜、修改后不验证 |
