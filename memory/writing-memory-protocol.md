# 写作记忆执行手册

**生效日期**: 2026-03-12  
**版本**: 2.0  
**状态**: 强制执行

---

## 核心原则

> **不写不存，不存不算完成。**

每一章写作完成后，必须**同时**完成两个动作：
1. **进度记录** → daily note
2. **正文落盘** → articles 目录

缺一不可。缺一就是**没完成**。

---

## 目录结构

```
memory/
├── YYYY-MM-DD.md              # 每日进度流水
├── project-index/
│   └── 项目名-summary.md       # 项目快照（关键设定、伏笔、人物状态）
└── articles/
    └── 项目名/
        ├── YYYY-MM-DD-第X章-第Y节.md   # 每节正文，按日期命名
        └── ...
```

---

## 强制流程

### 写作前（续写）
```
1. 读取 memory/project-index/项目名-summary.md（了解当前设定和伏笔）
2. 读取 articles/项目名/最近1-2章.md（保持文风连贯）
3. 开始续写
```

### 写作后（完成一节）
```
1. 【必须】将本节完整正文写入 articles/项目名/YYYY-MM-DD-第X章-第Y节.md
2. 【必须】更新 memory/YYYY-MM-DD.md 记录进度（本节关键剧情）
3. 【可选】如本节有关键伏笔/人物状态变更，更新 summary
4. 【验证】确认文件已落盘（ls 检查）
```

---

## 检查清单（每节完成后勾选）

- [ ] 正文已写入 `articles/项目名/日期-第X章-第Y节.md`
- [ ] 文件大小 > 0（确认不是空文件）
- [ ] daily note 已记录本节关键剧情
- [ ] 如关键伏笔/状态变更，summary 已更新
- [ ] 本节字数已统计

---

## 命名规范

**正文文件**（按节落盘）:
```
YYYY-MM-DD-第X章-第Y节.md
```

示例:
- `2026-03-10-第四章-第一节.md`
- `2026-03-10-第五章-第三节-沙海行舟.md`
- `2026-03-12-第九章-第四节.md`  ← 当前待写

---

## 禁止事项

❌ **只记进度，不存正文**  
❌ **只存正文，不更新 summary**  
❌ **延迟存档**（必须当时完成，每节一存）  
❌ **覆盖旧文件不备份**  
❌ **攒多节一起存**（必须每节独立文件）

---

## 故障处理

**如果发现缺失章节**:
1. 立即停止写作
2. 向 aur 报告缺失的章节
3. 请求 aur 提供原文补录
4. 补录完成后再继续

**如果磁盘/系统故障**:
1. 立即通知 aur
2. 从 aur 的本地备份恢复
3. 验证完整性

---

## 验证机制

**Heartbeat 检查时**:
```bash
# 检查每个活跃项目的 articles 目录
ls -la memory/articles/项目名/

# 确认最近3天是否有新文件
find memory/articles/项目名/ -mtime -3 -type f
```

**发现问题立即报告**:
> "aur，检测到《XXX》项目最近3天没有新的章节文件落盘，是否遗漏？"
