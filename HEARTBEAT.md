# HEARTBEAT.md - PCEC 主动维护协议

**P**roactive **C**ontinuous **E**volution & **C**onsolidation

---

## 触发频率
- **Heartbeat**: 每小时检查（轻量）
- **深度维护**: 每天一次（归档模式提取）

---

## 每小时检查项（轻量）

### 1. 系统状态检查
- [ ] 网关状态: `openclaw gateway status`
- [ ] 未提交变更: `git status --porcelain`
- [ ] 如有异常，通知 aur

### 2. 记忆归档（快速扫描）
- [ ] 检查 `memory/` 是否有新文件
- [ ] 检查今日 daily note 是否创建
- [ ] 识别值得提炼到 MEMORY.md 的事件

---

## 每日深度维护（一次）

### 记忆整理流程
```
1. 读取近3天的 daily notes
2. 提取关键决策/观察/模式
3. 更新 MEMORY.md 对应章节
4. 更新项目档案进度
5. 如有新能力，更新 capability-tree.md
```

### 检查清单
- [ ] **项目状态**: 检查各项目待办进度
- [ ] **观察更新**: aur 的新偏好/习惯
- [ ] **技能沉淀**: 有新方法论？写入 capability-tree
- [ ] **备份**: git commit & push

---

## 记忆归档规则

**什么值得写入 MEMORY.md：**
- ✅ 重要决策及原因
- ✅ 发现的 aur 行为模式
- ✅ 项目里程碑/状态变更
- ✅ 新技能/工具掌握
- ✅ 协作边界调整

**什么留在 daily notes：**
- 日常对话流水
- 临时想法
- 未完成的事项

---

## 备份策略
- **本地**: 自动（workspace git）
- **远程**: 推送到配置的仓库
- **验证**: 维护后检查文件完整性

---

## 时效性权重（检索时应用）
- 7天内：1.5x 优先级
- 30天内：1.2x 优先级
- 90天+：常规权重
