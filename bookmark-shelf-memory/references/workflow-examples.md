# 完整工作流程示例

## 场景：开始一个新项目《星际穿越者》

### 第一步：初始化项目

运行脚本或手动创建：

```bash
mkdir -p "memory/articles/星际穿越者"
touch "memory/project-index/星际穿越者-summary.md"
```

编辑 summary 文件，填入：
- 项目类型：科幻/星际/冒险
- 核心设定：主角、世界观
- 初始剧情节点

### 第二步：日常写作

**Day 1 - 写第一章**

用户发送正文，AI 同时执行：

1. **主记忆记录**（写入 `memory/2026-03-09.md`）：
   ```markdown
   ## [项目] 《星际穿越者》第一章完成
   
   - 写了第一章《觉醒》
   - 日期: 2026-03-09
   ```

2. **正文存档**（写入 `memory/articles/星际穿越者/2026-03-09-第一章.md`）：
   - 完整正文内容

3. **更新快照**（编辑 `memory/project-index/星际穿越者-summary.md`）：
   - 添加第一章到剧情节点表
   - 记录新人物登场
   - 记录埋下的伏笔

**Day 2 - 写第二章**

同上流程。

### 第三步：跨天续写

**Day 3 - 用户说"继续写《星际穿越者》"**

AI 执行：

1. **读取快照**：
   ```bash
   cat memory/project-index/星际穿越者-summary.md
   ```
   - 获得：核心设定、人物状态、当前进度、待填伏笔

2. **读取最近章节**：
   ```bash
   ls -t memory/articles/星际穿越者/ | head -2
   # 输出: 2026-03-10-第二章.md, 2026-03-09-第一章.md
   ```
   - 读取第二章（文风连贯）

3. **不读取**：第一章及更早内容

4. **开始续写第三章**

5. **写完后更新**：
   - 主记忆添加书签
   - 创建第三章正文存档
   - 更新项目快照

### 第四步：查询历史

**用户问"昨天写了什么"**

AI 执行：

```bash
grep "《星际穿越者》" memory/2026-03-10.md
# 输出: 写了《星际穿越者》第二章
```

返回：昨天的书签记录。

**用户说"给我看第二章"**

AI 执行：

```bash
cat "memory/articles/星际穿越者/2026-03-10-第二章.md"
```

返回：完整正文。

### 第五步：多项目并行

同时写《星际穿越者》和《古代搞钱》：

```
memory/articles/
├── 星际穿越者/
│   ├── 2026-03-09-第一章.md
│   └── 2026-03-10-第二章.md
└── 穿越之我在古代搞钱/
    └── 2026-03-10-第一章.md
```

每个项目独立快照，互不影响。

## 故障排查

**问题：续写时人物状态不对**
- 检查：summary 文件是否及时更新
- 解决：每章写完后立即更新人物状态

**问题：找不到某章节**
- 检查：主记忆书签是否记录正确
- 检查：文件名是否符合 `日期-章节名.md` 格式
- 检查：是否在正确的项目目录下

**问题：搜索结果太多干扰**
- 确保：grep 时排除 `memory/articles/` 目录
- 使用：`grep -r "关键词" memory/*.md memory/project-index/*.md`
