# 书签+书架记忆管理

## 核心概念

**问题**：AI 每次对话都是"空白"醒来，跨天续写时难以保持设定连贯，全文重读又太消耗 token。

**解决方案**：三层分离架构
- **书签（主记忆）**：轻量记录，只存"写了什么"
- **书架（正文存档）**：详细内容，按需加载
- **项目快照（summary）**：关键设定、人物状态、伏笔，续写时必读

## 目录结构

```
memory/
├── 2026-03-09.md                    # 主记忆（书签）- 每日对话记录
├── project-index/                   # 项目快照
│   ├── 富可敌国的我-summary.md      # 关键设定、进度、伏笔
│   ├── 穿越之我在古代搞钱-summary.md
│   └── ...
└── articles/                        # 书架（正文存档）
    ├── 富可敌国的我/
    │   ├── 2026-03-08-第十四章.md
    │   └── 2026-03-09-第十五章.md
    ├── 穿越之我在古代搞钱/
    └── ...
```

## 工作流程

### 1. 写作时

**用户发送正文** → AI 同时执行：
1. 在主记忆（daily note）中记录：`写了《XXX》第X章，日期`
2. 创建正文存档：`memory/articles/项目名/日期-章节名.md`
3. 更新项目快照：`memory/project-index/项目名-summary.md`
   - 添加新进度
   - 更新人物状态
   - 记录新伏笔

### 2. 续写时

**用户说"继续写《XXX》"** → AI 执行：
1. 读取 `memory/project-index/XXX-summary.md`（关键设定、人物状态、伏笔）
2. 读取 `memory/articles/XXX/` 下最近 1-2 章正文（文风连贯）
3. **不读**更早的章节
4. 开始续写

### 3. 查询时

**用户问"昨晚写了什么"** → AI 执行：
1. 搜索主记忆文件（grep `memory/*.md`）
2. 返回列表：`《富可敌国的我》第十四章 — 2026-03-08`
3. 用户选择后，根据日期+章节名读取正文存档

## 项目快照（summary）格式

```markdown
# 项目名称 - 项目快照

**项目类型**: 类型/风格/标签  
**当前进度**: 已完成X章，等待继续第X章  
**更新时间**: YYYY-MM-DD

---

## 核心设定

**男主**: 姓名
- 身份：
- 性格：
- 当前状态：

**关键人物**:
- 人物A：关系、状态
- 人物B：关系、状态

**世界观**: 关键规则

---

## 剧情节点

| 章节 | 核心事件 | 伏笔/状态 |
|------|----------|-----------|
| 第1-3章 | 事件A | ✅ 完成 |
| 第4-6章 | 事件B | ✅ 完成 |
| 第7章 | 事件C | ⏳ 待续写 |

---

## 待填伏笔

- [ ] 伏笔A
- [ ] 伏笔B

---

## 写作要求（已确认）

> **节奏原则**: 具体写作要求  
> **来源**: aur YYYY-MM-DD 反馈

---

## 文件位置

- 大纲: `memory/project-XXX-大纲.md`
- 人物卡: `memory/project-XXX-人物卡.md`
- 各章节正文: `memory/articles/XXX/`
```

## 文件命名规则

**正文存档**：`YYYY-MM-DD-章节名.md`
- 示例：`2026-03-08-第十四章.md`

**项目快照**：`项目名-summary.md`
- 示例：`富可敌国的我-summary.md`

**目录结构**：`memory/articles/项目名/`
- 示例：`memory/articles/富可敌国的我/`

## 搜索策略

**默认搜索**（日常查询）：
```bash
grep -r "关键词" memory/*.md memory/project-index/*.md
```
**不搜索** `memory/articles/`（避免正文污染结果）

**按需读取**（用户确认后）：
- 根据日期和章节名构造路径
- 读取 `memory/articles/项目名/日期-章节名.md`

## 初始化新项目

当开始新写作项目时：

1. 创建目录：`mkdir -p memory/articles/项目名`
2. 创建快照：`memory/project-index/项目名-summary.md`
   - 填入项目类型、核心设定、初始剧情节点
3. 在主记忆中记录项目启动

## 更新项目快照

每写完一章，更新 summary：

1. **进度**：添加新章节到剧情节点表
2. **人物状态**：更新各人物最新状态/位置
3. **伏笔**：记录新埋下的伏笔，标记已回收的伏笔
4. **特殊要求**：如有新的写作要求，添加到"写作要求"区块

## 跨天续写检查清单

- [ ] 已读取 project-index/项目-summary.md
- [ ] 已读取 articles/项目/最近1-2章.md
- [ ] 已确认当前人物状态和位置
- [ ] 已确认待填伏笔
- [ ] 已回顾写作要求

## 迁移旧项目

如有存量文章未按此机制存储：

1. 为每个项目创建 summary 文件，汇总关键信息
2. （可选）将存量章节移动到 `articles/项目名/` 目录
3. 在主记忆中添加书签记录

---

**核心原则**：轻量书签可搜索，详细正文按需读，关键设定不丢失。
